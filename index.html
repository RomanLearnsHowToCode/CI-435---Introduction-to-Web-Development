Week 10: CSS position property

Published on December 3, 2018

Lecture about CSS and also talking about coursework

1. Normal flow HTML document
Elements are displayed in the order in which they are written in the HTML, we can also change or arrange them to show in a different order with CSS. Block level elements start on a new line, while inline elements are nested within block elements.

2. Float and Clear properties
The CSS float and clear properties are used to take an element out of normal flow and position it to the right or left of its containing (parent) box, the element that follows around the floated element until the clear property ends the float.

3. Position property - static, relative, absolute, fixed, sticky
CSS position property is way how to remove elements from normal flow and position them anywhere on the page. There is also an amazing puzzle solving game, which can help a student to learn and understand CSS positioning: Flex box froggy game

4. Offset properties - top, right, bottom, left

5. Z-index property
The z-index property controls the stacking order of elements. Whether elements stay on top or lay behind other elements. The z-index value is an integer.

6. Common use cases for the position property

7. CSS3 colour - RGBA, Alpha channel
CSS colour can be specified by both: key-words or 6 digit hexadecimal code. There is a tool from Adobe Creative Cloud to help us with colour choice and code. Colour Wheel

8. Browser support
Macromedia Dreamweaver comes with a tool, to help us optimize our website or preview it in different browser support, it can help us to debug or set different rules in CSS for better user experience.

Week 8: Responsive Web Design 3

Published on November 19, 2018

CSS3 responsive layout

1. Basic layout properties(float, clean and position)
Web page content is presented in the order in which it is marked up in HTML document unless the normal flow is disrupted by CSS declaring layout properties. In CSS we can set rules for various tags and elements, we can even createnew classes.

2. Review of RWD core techniques

3. CSS3 layout modules
CSS3 has 3 modules for layout: Grid layout (Layout in 2 dimensions in both columns and rows), Multi-column layout (To split content into columns) Flexible box layout (Layout in one dimension – either in columns or rows). Each layout have it cons and pros. They are also suitable for different purpose.

4. Content-led design - media query breakpoints
We should first write the content of our website, then work on a design. Media queries should be written in the style sheet, so it will override the default style.

5. How to hide elements
Sometimes we want to hide content. There are multiple ways how to hide content with style sheet elements, some hidden elements will still affect layout, so we have to pick the right code option.

Week 5: The CSS box

Published on October 29, 2018

Model and measurement of CSS and styling text.

1. The cascade - "specificity".
The cascade is how browsers determine which rule or rules apply to which element – and how to handle conflicts between rules. The browser style sheet is a default, an external style sheet over-rides browser CSS because you set rules for it.

2. CSS box model.
Controls how elements are displayed and positioned on a web page, every HTML element is treated as if it is a box. Boxes can be organized in a sequence, one after another, or be nested within other boxes. There are block and inline levels of boxes.

3. CSS - absolute and relative lenght units.
Absolute units are inches, centimeters, millimeters, points. Relative units are screen pixels, percent, root element font size, element font size, viewport width. And each unit has got different values and behavior.

4. Block and inline elements.

5. Viewport percentage lenghts (vw,vh).
Viewport-percentage lengths define the length of an element relative to the size of the viewport, the visible portion of the web page.

Week 4: Cascading Stylesheet

Published on October 22, 2018

In fourth lecture we started with CSS.

1. What is CSS?
Cascading style sheet is a declarative language for specifying the presentation of HTML elements by a browser. So how do I understand this statement? Well CSS is a very useful tool (code), which can help us to with stylisation of our whole website by one file. With CSS you can set rules for each section of code, create new classes etc.

2. CSS syntax - rules, properties and values
Elements are objects in the HTML document. CSS rules target elements in order to instruct the browser, how to present content.CSS consist of a number of rules. A rule consist of two parts, the selector which targets an HTML element and one or more declarations, inside curly braces, state how the element will be displayed. A declaration also has two parts, a property – positon, color, margin etc. And a value for the property – e.g. margin: 1.2em

3. Selectors
Selectors are targets of elements, for example caption, time, img, h1-h6

4. Attaching an external stylesheet to all web pages
Each web page must have in head tag link for stylesheet, each web page can be linked to different stylesheet, but it is better to keep website consistent to avoid confusion and misread for user.

5. Emedded and inline CSS.

6. Type, Class, Id and pseudo-class selectors.

7. Normalize.css - what is it, how does it works, how to link it with website.

Week 3: HTML5

Published on October 15, 2018

Document structure & hypertext

1. Section elements, and why HTML 5 is easier and better to write.
HTML5 document starts with doctype declaration at a very first line of webpage document. HTML5 is easier for web designers to write, and brings new functions. We don’t need to write div for each section elements anymore.

2. Index.html document object model.
The structure of section elements in meaning full order. Ex. Footer is at the bottom of a page, and a header is on the top, even though you can display sections in a different order with CSS grid layout, it is a rule how website/page should be written.

3. Div elements - how did they changed with HTML 5 and where to use them.
Used only in order to mark-up grouped content in a document in order to style these elements with CSS properties.

4. Absolute and relative links, when, why and how to use them.
The way how I understand absolute and relative links is that with an absolute link you give complete destination of an object, while a relative link is often used only in the local website, for example, to link an image from the different directory.

5. What is viewport, how to set it correctly for multiple devices.
A viewport is an optimization tool for website, to display content for multiple devices in order we want.

6. Anchor tags.
The anchor tag is to link together two separate but related pages or pieces of information. Can link internal link, local link, an external link or email link.
